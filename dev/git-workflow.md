# Git Workflow

## Commit Format

```
type: emoji description
```

**Rules:**
- Present tense: "Add feature" not "Added"
- Imperative mood: "Fix bug" not "Fixes"
- 50 char limit first line
- No period at end
- **NEVER mention AI/Claude**

---

## Commit Types

| Type | Emoji | Use |
|------|-------|-----|
| feat | ✨ | New feature |
| fix | 🐛 | Bug fix |
| refactor | ♻️ | Restructure |
| style | 💄 | Formatting |
| docs | 📝 | Documentation |
| chore | 🔧 | Maintenance |
| test | ✅ | Tests |
| perf | ⚡ | Performance |
| checkpoint | 📍 | Stable state |

---

## Examples

```bash
✅ feat: ✨ add joystick controller component
✅ fix: 🐛 correct WebSocket reconnect logic
✅ refactor: ♻️ extract terrain chunk manager
✅ docs: 📝 update PLAN with phase 1b tasks
✅ chore: 🔧 configure biome for svelte files
✅ checkpoint: 📍 before SvelteKit migration

❌ Generated by Claude
❌ AI suggested changes
❌ feat: Added stuff.
```

## Branch Strategy

| Branch | Purpose |
|--------|---------|
| `main` | Stable, working state |
| `phase-X-name` | Feature branches per phase |

```bash
# New phase
git checkout -b phase-0-sveltekit-migration

# Merge when phase complete
git checkout main
git merge phase-0-sveltekit-migration
```

## Issue Tracking

**Key principle:** Don't fix problems immediately!

### Workflow

```
1. Discover problem during implementation
2. Create Issue:
   gh issue create --title "Bug: [desc]" --body "[details]"
   OR note in PLAN.md under "Backlog"
3. Continue with current task
4. Fix issues in dedicated session
```

### Why?

- Keeps sessions focused
- Prevents scope creep
- Creates documentation trail

## Safety

```bash
git stash push -m "description"  # Save
git stash pop                    # Restore
git checkout main                # Emergency return
```
