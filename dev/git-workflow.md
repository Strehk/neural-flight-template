# Git Workflow

## Commit Format

```
type: emoji description
```

**Rules:**
- Present tense: "Add feature" not "Added"
- Imperative mood: "Fix bug" not "Fixes"
- 50 char limit first line
- No period at end
- **NEVER mention AI/Claude**

---

## Commit Types

| Type | Emoji | Use |
|------|-------|-----|
| feat | ✨ | New feature |
| fix | 🐛 | Bug fix |
| refactor | ♻️ | Restructure |
| style | 💄 | Formatting |
| docs | 📝 | Documentation |
| chore | 🔧 | Maintenance |
| test | ✅ | Tests |
| perf | ⚡ | Performance |
| checkpoint | 📍 | Stable state |

---

## Examples

```bash
✅ feat: ✨ add VR scene with rotating cube
✅ fix: 🐛 resolve WebXR session error
✅ refactor: ♻️ extract scene setup to module
✅ checkpoint: 📍 before controller implementation

❌ Generated by Claude
❌ AI suggested changes
❌ feat: Added stuff.
```

## Feature Branch Workflow

```bash
# 1. Create branch
git checkout -b feature/[name]

# 2. Work & commit
git add .
git commit -m "type: emoji description"

# 3. Merge to main
git checkout main
git merge feature/[name]

# 4. Delete branch
git branch -d feature/[name]
```

---

## Issue Tracking

**Key principle:** Don't fix problems immediately!

### Workflow

```
1. Discover problem during implementation
2. Create Issue:
   gh issue create --title "Bug: [desc]" --body "[details]"
   OR note in PLAN.md under "Backlog"
3. Continue with current task
4. Fix issues in dedicated session
```

### Why?

- Keeps sessions focused
- Prevents scope creep
- Creates documentation trail

## Quick Reference

| Task | Command |
|------|---------|
| Status | `git status` |
| Checkpoint | `git add -A && git commit -m "checkpoint: 📍 [desc]"` |
| New branch | `git checkout -b feature/name` |
| Create issue | `gh issue create --title "..."` |

## Safety

```bash
git stash push -m "description"  # Save
git stash pop                    # Restore
git checkout main                # Emergency return
```
